# =============================================================================
# ROCm INSTALLATION NOTES
# =============================================================================
# 1. Do NOT `pip install -r requirements.txt` directly on a machine that
#    needs GPU inference (torch + vllm) without first installing the ROCm
#    builds, or pip will pull the CUDA/CPU wheels from PyPI.
#
# 2. Recommended order on the AMD Developer Cloud (ROCm 6.1) image:
#
#    pip install --no-cache-dir \
#      torch==2.4.0 \
#      --index-url https://download.pytorch.org/whl/rocm6.1
#
#    pip install --no-cache-dir vllm==0.5.4 \
#      --extra-index-url https://download.pytorch.org/whl/rocm6.1
#
#    pip install -r requirements.txt --no-deps  # for torch/vllm lines above
#    pip install -r requirements.txt            # install everything else normally
#
#    (Alternatively, use AMD's prebuilt vLLM ROCm Docker image as the base
#    image, which already ships torch+vllm for ROCm, and only install the
#    remaining packages from this file on top of it.)
#
# 3. faster-whisper / ctranslate2 also benefits from ROCm/HIP builds for GPU
#    decoding, but will fall back to CPU if not available - this is fine for
#    most deployments since Whisper inference can run on CPU or a separate
#    smaller GPU allocation.
